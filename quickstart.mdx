## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.12 or higher** - Check with `python --version`
- **PostgreSQL** - A running PostgreSQL database server
- **uv** (recommended) or **pip** - Python package manager

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the Resemblyzer App repository to your local machine:
    
    ```bash
    git clone <your-repository-url>
    cd resemblyzer-app
    ```
  </Step>

  <Step title="Install dependencies">
    The project uses `pyproject.toml` for dependency management. Install all required packages:
    
    <CodeGroup>
    ```bash uv (recommended)
    uv pip install -e .
    ```
    
    ```bash pip
    pip install -e .
    ```
    </CodeGroup>

    <Info>
    Key dependencies include:
    - **FastAPI** - Modern web framework for building APIs
    - **Resemblyzer** - Voice embedding library for speaker verification
    - **SQLAlchemy** - Database ORM
    - **Alembic** - Database migration tool
    - **psycopg** - PostgreSQL adapter
    </Info>
    
    <Warning>
    The code uses `python-dotenv` for loading environment variables, but it's not included in `pyproject.toml`. You'll need to install it separately:
    ```bash
    pip install python-dotenv
    ```
    </Warning>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root with your database connection:
    
    ```bash .env
    DATABASE_URL=postgresql://username:password@localhost:5432/resemblyzer_db
    ```

    <Warning>
    Replace `username`, `password`, and `resemblyzer_db` with your actual PostgreSQL credentials and database name.
    </Warning>
  </Step>

  <Step title="Create the database">
    Create a new PostgreSQL database for the application:
    
    ```bash
    psql -U postgres -c "CREATE DATABASE resemblyzer_db;"
    ```
    
    Or use your preferred PostgreSQL client to create the database.
  </Step>

  <Step title="Run database migrations">
    Apply the database schema using Alembic migrations:
    
    ```bash
    alembic upgrade head
    ```
    
    This will create the `users` table with columns for:
    - `id` - Primary key
    - `name` - User's name
    - `voice_emmbedding` - Binary storage for voice embeddings
  </Step>
</Steps>

## Running the Server

Start the FastAPI development server using uvicorn:

```bash
fastapi dev src/main.py
```

The API will be available at:
- **API**: http://localhost:8000
- **Interactive docs**: http://localhost:8000/docs
- **Alternative docs**: http://localhost:8000/redoc

<Tip>
The server runs with auto-reload enabled, so changes to your code will automatically restart the server.
</Tip>

## Testing the API

Now that your server is running, let's test the voice authentication endpoints.

<Steps>
  <Step title="Register a new user">
    Register a user with their voice sample. You'll need an audio file (WAV, MP3, or MPEG format):
    
    ```bash
    curl -X POST "http://localhost:8000/users/register" \
      -F "name=John Doe" \
      -F "audio=@/path/to/voice_sample.wav"
    ```
    
    **Response:**
    ```json
    {
      "message": "User created successfully",
      "user_id": 1,
      "name": "John Doe",
      "audio": "voice_sample.wav",
      "audio_type": "audio/wav",
      "embed": [0.123, 0.456, ...]
    }
    ```

    <Info>
    The API processes the audio file using Resemblyzer to create a unique voice embedding that's stored in the database.
    </Info>
  </Step>

  <Step title="Authenticate with voice">
    Verify a user's identity by comparing their voice against the stored embedding:
    
    ```bash
    curl -X POST "http://localhost:8000/users/login" \
      -F "name=John Doe" \
      -F "audio=@/path/to/login_voice.wav"
    ```
    
    **Success response (similarity ≥ 0.80):**
    ```json
    {
      "status": "success",
      "message": "Bienvenido John Doe",
      "similarity": 0.8542
    }
    ```
    
    **Failed authentication (similarity < 0.80):**
    ```json
    {
      "detail": "Acceso denegado. Similitud: 0.7234"
    }
    ```

    <Note>
    The authentication threshold is set at **0.80 similarity**. Adjust this in `src/routers/users.py:69` if needed.
    </Note>
  </Step>

  <Step title="List all users">
    Retrieve all registered users:
    
    ```bash
    curl "http://localhost:8000/users/users"
    ```
    
    **Response:**
    ```json
    [
      {
        "id": 1,
        "name": "John Doe"
      },
      {
        "id": 2,
        "name": "Jane Smith"
      }
    ]
    ```
  </Step>
</Steps>

## Understanding the Code

Here's how the key components work together:

### FastAPI Application (`src/main.py`)

```python src/main.py
from fastapi import FastAPI
from .routers import users
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

# CORS configuration for local development
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "http://localhost:8080"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Include user authentication routes
app.include_router(users.router)
```

### Database Configuration (`src/database.py`)

```python src/database.py
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
import os

DATABASE_URL = os.getenv("DATABASE_URL")

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

### Voice Embedding Model (`src/models.py`)

```python src/models.py
from sqlalchemy import Column, Integer, String, LargeBinary

class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True)
    name = Column(String)
    voice_emmbedding = Column(LargeBinary)  # Stores numpy array as bytes
```

## Troubleshooting

### Database Connection Issues

If you encounter database connection errors:

1. Verify PostgreSQL is running: `pg_isready`
2. Check your `DATABASE_URL` in `.env`
3. Ensure the database exists: `psql -l`

### Audio Format Errors

The API accepts these audio formats:
- `audio/wav`
- `audio/mp3`
- `audio/mpeg`

If you get "Formato de audio invalido", check your file's MIME type.

### Import Errors

If you see module import errors, ensure you installed the package in editable mode:

```bash
pip install -e .
```

This makes the `src` package importable.
