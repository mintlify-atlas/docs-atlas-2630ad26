## Environment Variables

### Required Configuration

Set these environment variables in your production environment:

```bash
# Database connection
DATABASE_URL=postgresql://user:password@host:5432/dbname?sslmode=require

# Application settings
ENVIRONMENT=production
DEBUG=false

# Security (if implemented)
SECRET_KEY=your-secret-key-here
ALLOWED_ORIGINS=https://yourdomain.com,https://app.yourdomain.com
```

<Warning>
Never use development database credentials in production. Always use strong, unique passwords and enable SSL.
</Warning>

### Database URL Configuration

For production, your `DATABASE_URL` should include security options:

<CodeGroup>
```env Managed PostgreSQL (Heroku, Render, etc.)
DATABASE_URL=postgresql://user:pass@host.provider.com:5432/dbname?sslmode=require
```

```env Self-hosted with SSL
DATABASE_URL=postgresql://user:pass@host:5432/dbname?sslmode=require&sslrootcert=/path/to/ca-cert.pem
```

```env Connection Pooling (PgBouncer)
DATABASE_URL=postgresql://user:pass@pooler.host.com:6432/dbname?sslmode=require
```
</CodeGroup>

## Database Connection Pooling

For production workloads, configure SQLAlchemy connection pooling:

```python src/database.py
import os
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker
from dotenv import load_dotenv

load_dotenv()

DATABASE_URL = os.getenv("DATABASE_URL")

# Production configuration with connection pooling
engine = create_engine(
    DATABASE_URL,
    pool_size=10,              # Maximum number of connections to maintain
    max_overflow=20,           # Allow 20 additional connections during load
    pool_pre_ping=True,        # Verify connections before using
    pool_recycle=3600,         # Recycle connections after 1 hour
    echo=False,                # Disable SQL logging in production
)

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
```

<Info>
Adjust `pool_size` and `max_overflow` based on your expected concurrent users. Each worker process will maintain its own connection pool.
</Info>

## Running with Uvicorn

### Development vs Production

<CodeGroup>
```bash Development
# Single worker with auto-reload
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

```bash Production
# Multiple workers for concurrent requests
uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 4
```
</CodeGroup>

### Worker Configuration

Calculate the number of workers based on CPU cores:

```bash
# Formula: (2 x CPU cores) + 1
# For 2 cores: 5 workers
uvicorn src.main:app --host 0.0.0.0 --port 8000 --workers 5
```

<Warning>
Each worker loads the Resemblyzer VoiceEncoder model into memory (~100MB). Monitor memory usage when scaling workers.
</Warning>

### Using Gunicorn with Uvicorn Workers

For production, use Gunicorn with Uvicorn workers for better process management:

```bash
# Install gunicorn
pip install gunicorn

# Run with uvicorn workers
gunicorn src.main:app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --timeout 120 \
  --access-logfile - \
  --error-logfile -
```

<Accordion title="Gunicorn Configuration File">
Create `gunicorn.conf.py` in your project root:

```python gunicorn.conf.py
import multiprocessing
import os

# Server socket
bind = "0.0.0.0:8000"

# Worker processes
workers = int(os.getenv("WEB_CONCURRENCY", multiprocessing.cpu_count() * 2 + 1))
worker_class = "uvicorn.workers.UvicornWorker"

# Timeout for workers
timeout = 120
graceful_timeout = 30

# Logging
accesslog = "-"
errorlog = "-"
loglevel = "info"

# Process naming
proc_name = "resemblyzer-app"
```

Run with:
```bash
gunicorn -c gunicorn.conf.py src.main:app
```
</Accordion>

## CORS Configuration

Update CORS settings for your production domain:

```python src/main.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
import os

app = FastAPI()

# Production CORS configuration
origins = [
    "https://yourdomain.com",
    "https://app.yourdomain.com",
]

# Allow environment variable override
if os.getenv("ALLOWED_ORIGINS"):
    origins = os.getenv("ALLOWED_ORIGINS").split(",")

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

<Warning>
Never use `allow_origins=["*"]` in production. Always specify exact domains.
</Warning>

## Audio File Handling

### Temporary File Cleanup

The application uses temporary files for audio processing. Current implementation:

```python src/routers/users.py
import tempfile
import shutil
from pathlib import Path

@router.post("/register")
async def create_users(
    name: str = Form(...),
    audio: UploadFile = File(...),
    db: Session = Depends(database.get_db),
):
    # Save uploaded file to temporary location
    with tempfile.NamedTemporaryFile(delete=False, suffix=".wav") as tmp:
        shutil.copyfileobj(audio.file, tmp)
        tmp_path = Path(tmp.name)
    
    try:
        # Process audio
        wav_audio = preprocess_wav(tmp_path)
        embed = encoder.embed_utterance(wav_audio)
        
        # Store in database
        user = models.User(name=name, voice_emmbedding=embed.tobytes())
        db.add(user)
        db.commit()
    finally:
        # Always cleanup temporary file
        if tmp_path.exists(): 
            tmp_path.unlink()
```

<Info>
Temporary files are automatically cleaned up in the `finally` block, even if an error occurs during processing.
</Info>

## Security Considerations

### HTTPS Configuration

<Steps>
  <Step title="Use a Reverse Proxy">
    Deploy behind Nginx or Caddy to handle SSL termination:
    
    ```nginx nginx.conf
    server {
        listen 443 ssl http2;
        server_name api.yourdomain.com;
        
        ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;
        
        location / {
            proxy_pass http://127.0.0.1:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
    ```
  </Step>
  
  <Step title="Configure Trust Proxy Headers">
    Tell FastAPI to trust proxy headers:
    
    ```python src/main.py
    from fastapi import FastAPI
    
    app = FastAPI(
        docs_url="/docs" if os.getenv("ENVIRONMENT") == "development" else None,
        redoc_url="/redoc" if os.getenv("ENVIRONMENT") == "development" else None,
    )
    ```
  </Step>
</Steps>

### Voice Embedding Storage

Voice embeddings are sensitive biometric data:

<Warning>
**Important Security Measures:**
- Store embeddings encrypted at rest (use database-level encryption)
- Use secure database connections (SSL/TLS)
- Implement access controls and audit logging
- Consider GDPR/privacy regulations for biometric data
- Provide user data deletion capabilities
</Warning>

### Authentication Threshold

The similarity threshold for voice authentication is defined in `src/routers/users.py:69`:

```python
UMBRAL = 0.80  # 80% similarity required
if similarity < UMBRAL:
    raise HTTPException(status_code=401, detail=f"Access denied. Similarity: {similarity:.4f}")
```

<Tip>
Adjust the threshold based on your security requirements:
- **Higher threshold (0.85-0.90)**: More secure, may reject valid users
- **Lower threshold (0.70-0.75)**: More permissive, may accept imposters

Test with real users to find the optimal balance.
</Tip>

## Running Migrations in Production

### Pre-deployment Migration

Always run migrations before deploying new code:

```bash
# In your deployment script
alembic upgrade head
```

### Zero-downtime Migrations

For critical production systems:

<Steps>
  <Step title="Deploy Backward-Compatible Schema Changes">
    Add new columns as nullable first:
    ```python
    op.add_column('users', sa.Column('new_field', sa.String(), nullable=True))
    ```
  </Step>
  
  <Step title="Deploy Application Code">
    Update application to use new fields while maintaining compatibility.
  </Step>
  
  <Step title="Backfill Data">
    Populate new fields for existing rows:
    ```python
    op.execute("UPDATE users SET new_field = 'default' WHERE new_field IS NULL")
    ```
  </Step>
  
  <Step title="Make Non-nullable">
    Once data is backfilled, enforce constraints:
    ```python
    op.alter_column('users', 'new_field', nullable=False)
    ```
  </Step>
</Steps>

## Deployment Platforms

### Docker Deployment

<Accordion title="Dockerfile Example">
```dockerfile Dockerfile
FROM python:3.12-slim

WORKDIR /app

# Install system dependencies for audio processing
RUN apt-get update && apt-get install -y \
    libsndfile1 \
    ffmpeg \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY pyproject.toml .
RUN pip install --no-cache-dir -e .

# Copy application code
COPY . .

# Run migrations and start server
CMD alembic upgrade head && \
    gunicorn src.main:app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8000
```
