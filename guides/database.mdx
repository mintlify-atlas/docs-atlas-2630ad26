## PostgreSQL Setup

The Resemblyzer App uses PostgreSQL to store user information and voice embeddings.

### Installing PostgreSQL

<CodeGroup>
```bash macOS (Homebrew)
brew install postgresql@15
brew services start postgresql@15
```

```bash Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

```bash Windows
# Download installer from https://www.postgresql.org/download/windows/
# Or use Chocolatey:
choco install postgresql
```
</CodeGroup>

### Create Database

Create a database for the application:

```bash
# Using psql
psql -U postgres
CREATE DATABASE resemblyzer_db;
\q
```

Or using the `createdb` command:

```bash
createdb -U postgres resemblyzer_db
```

## Database Connection

### Connection String Format

Configure your database connection in the `.env` file:

```env .env
DATABASE_URL=postgresql://username:password@host:port/database
```

### Example Configurations

<CodeGroup>
```env Local Development
DATABASE_URL=postgresql://postgres:password@localhost:5432/resemblyzer_db
```

```env Docker Container
DATABASE_URL=postgresql://postgres:password@db:5432/resemblyzer_db
```

```env Production (with SSL)
DATABASE_URL=postgresql://user:pass@prod-host.com:5432/resemblyzer_db?sslmode=require
```
</CodeGroup>

<Warning>
Never commit your `.env` file to version control. It's already included in `.gitignore`.
</Warning>

### Database Configuration in Code

The application uses SQLAlchemy for database operations. Connection is configured in `src/database.py`:

```python src/database.py
import os
from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker
from dotenv import load_dotenv

load_dotenv()

DATABASE_URL = os.getenv("DATABASE_URL")

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

## Database Schema

### Users Table

The main table stores user information and voice embeddings:

```python src/models.py
from sqlalchemy import Column, Integer, String, LargeBinary
from .database import Base

class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True)
    name = Column(String)
    voice_emmbedding = Column(LargeBinary)
```

### Schema Overview

| Column | Type | Description |
|--------|------|-------------|
| `id` | Integer | Primary key, auto-incrementing |
| `name` | String | User's name (used for login) |
| `voice_emmbedding` | LargeBinary | Serialized numpy array of voice embedding (256 floats) |

<Info>
Voice embeddings are stored as binary data using numpy's `tobytes()` method and reconstructed with `np.frombuffer()` during authentication.
</Info>

## Alembic Migrations

The project uses Alembic for database schema version control.

### Configuration

Alembic is configured in `alembic.ini` and `migrations/env.py`:

```ini alembic.ini
[alembic]
script_location = %(here)s/migrations
prepend_sys_path = .
path_separator = os

# Database URL is loaded from environment
sqlalchemy.url = 
```

The `migrations/env.py` file loads the database URL from your `.env` file:

```python migrations/env.py
import os
from dotenv import load_dotenv
from src.models import Base

load_dotenv()
database_url = os.getenv("DATABASE_URL")

if not database_url:
    raise ValueError("DATABASE_URL not found in environment")

config.set_main_option("sqlalchemy.url", database_url)
target_metadata = Base.metadata
```

### Migration History

The project has two migrations:

<Steps>
  <Step title="Initial Schema (643db261e5ab)">
    Creates the users table with initial columns:
    
    ```python migrations/versions/643db261e5ab_initial_schema.py
    def upgrade() -> None:
        op.create_table('users',
        sa.Column('id', sa.Integer(), nullable=False),
        sa.Column('name', sa.String(), nullable=True),
        sa.Column('email', sa.String(), nullable=True),
        sa.Column('embed', sa.String(), nullable=True),
        sa.PrimaryKeyConstraint('id'),
        sa.UniqueConstraint('email')
        )
    ```
  </Step>

  <Step title="Voice Embedding Update (2fd98754c923)">
    Updates the schema for proper voice embedding storage:
    
    ```python migrations/versions/2fd98754c923_user_voice_emmbedding.py
    def upgrade() -> None:
        # Add binary column for voice embeddings
        op.add_column('users', sa.Column('voice_emmbedding', sa.LargeBinary(), nullable=True))
        # Remove unused email column and string embed
        op.drop_constraint(op.f('users_email_key'), 'users', type_='unique')
        op.drop_column('users', 'email')
        op.drop_column('users', 'embed')
    ```
    
    <Info>
    This migration changes the embedding storage from String to LargeBinary for efficient binary data storage.
    </Info>
  </Step>
</Steps>

### Running Migrations

<Accordion title="Apply All Migrations">
```bash
alembic upgrade head
```
Upgrades the database to the latest version.
</Accordion>

<Accordion title="Check Current Version">
```bash
alembic current
```
Shows the current migration revision.
</Accordion>

<Accordion title="View Migration History">
```bash
alembic history
```
Displays all available migrations:
```
643db261e5ab -> 2fd98754c923 (head), user_voice_emmbedding
<base> -> 643db261e5ab, initial_schema
```
</Accordion>

<Accordion title="Rollback Migration">
```bash
# Downgrade one version
alembic downgrade -1

# Downgrade to specific revision
alembic downgrade 643db261e5ab

# Downgrade all migrations
alembic downgrade base
```
</Accordion>

### Creating New Migrations

When you modify models, create a new migration:

<Steps>
  <Step title="Auto-generate Migration">
    ```bash
    alembic revision --autogenerate -m "description of changes"
    ```
    
    Alembic will detect changes in your models and generate a migration file.
  </Step>

  <Step title="Review Generated Migration">
    Check the generated file in `migrations/versions/` and verify the changes are correct.
    
    <Warning>
    Always review auto-generated migrations. Alembic may not detect all changes correctly.
    </Warning>
  </Step>

  <Step title="Apply the Migration">
    ```bash
    alembic upgrade head
    ```
  </Step>
</Steps>

## Database Operations in Code

### Dependency Injection

FastAPI uses dependency injection to provide database sessions:

```python
from fastapi import Depends
from sqlalchemy.orm import Session
from src.database import get_db

@router.post("/register")
async def create_user(
    name: str = Form(...),
    audio: UploadFile = File(...),
    db: Session = Depends(get_db),
):
    # db session is automatically provided and closed
    user = models.User(name=name, voice_emmbedding=embed.tobytes())
    db.add(user)
    db.commit()
    db.refresh(user)
    return user
```

### Binary Data Storage

Voice embeddings are stored as binary data:

```python src/routers/users.py
import numpy as np

# Store embedding (registration)
embed = encoder.embed_utterance(wav_audio)  # Returns numpy array
user = models.User(
    name=name,
    voice_emmbedding=embed.tobytes()  # Convert to bytes
)
db.add(user)
db.commit()

# Retrieve embedding (authentication)
user = db.query(models.User).filter(models.User.name == name).first()
master_embed = np.frombuffer(user.voice_emmbedding, dtype=np.float32)
```

<Tip>
The voice embeddings from Resemblyzer are 256-dimensional float32 arrays. When stored as binary, each embedding takes approximately 1KB of storage.
</Tip>

## Database Inspection

### Using psql

Connect to your database:

```bash
psql -U postgres -d resemblyzer_db
```

Useful commands:

```sql
-- List all tables
\dt

-- Describe users table
\d users

-- Query users
SELECT id, name, length(voice_emmbedding) as embedding_size FROM users;

-- Count users
SELECT COUNT(*) FROM users;
```

### Using Python

Query the database directly:

```python
from src.database import SessionLocal
from src.models import User

db = SessionLocal()
users = db.query(User).all()
for user in users:
    print(f"User {user.id}: {user.name}")
db.close()
```
